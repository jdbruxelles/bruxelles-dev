Êtes-vous un.e développeur.se (web ou autre) basé.e à Bruxelles et à la recherche d'un espace Web gratuit pour héberger votre portfolio ou votre autre projet ?

Consultez GitHub ou Netlify pour en savoir plus. Pour que tout soit parfait, je vous fourni une URL libre, élégante et qui correspond à votre domaine et zone d'activité comme indiqué dans les exemples ci-dessus.

Vous êtes prêt.e à avoir votre place sur bruxelles.dev ? Faites votre demande via GitHub ou contactez-moi directement.

Important En tant que propriétaire du projet, vous gardez un contrôle total sur votre contenu publié. Cela signifie également que tous les droits et obligations liés à la publication d'une page GitHub (par exemple, RGPD) restent sous votre entière responsabilité.

Si vous avez opté pour le service Netlify ou un autre que GitHub Page, veuillez me contacter directement. Si non, si vous avez opté pour Github Pages, veuillez suivre les instructions suivantes :

Si vous ne l'avez pas déjà fait, il est temps de vous connecter à votre compte GitHub et de configurer votre GitHub Page en suivant les instructions fournies ici. Pour prendre de l'avance, vous pouvez simplement utiliser le générateur avec l'un des thèmes fournis et ajouter du contenu à votre nouvelle page.

Vous devez maintenant déterminer votre sous-domaine bruxelles.dev : choisissez votre nom d'utilisateur, votre prénom, votre nom ou le nom de votre projet en tenant compte des URLs existantes (pour "https://foo.github.io/bar", "foo.bruxelles.dev" ou "bar.bruxelles.dev" est possible).

Ajoutez un fichier nommé "CNAME" à votre projet (dans la branche "gh-pages" pour les pages de projet) avec une seule ligne correspondant au domaine que vous avez choisi. (par exemple "foo.bruxelles.dev" sans guillemets). Si vous rencontrez des problèmes, consultez la section sur les URL personnalisées dans l'aide de GitHub Pages.

Pour terminer la procédure, faites un pull request dans mon project GitHub qui ajoute votre sous-domaine à la liste des domaines bruxelles.dev existants. Votre nouvelle URL devrait être disponible dans les 24 heures (gardez un œil sur votre pull request en cas de conflit de nom ou toute autre question de ma part).
